FAQ

Do you store my keys?

No. Your Supabase connection string and secret key stay in your browser tab and are sent directly to your own projects for the transfer. Nativize never stores or logs them.

Does it really move my users without password resets?

Yes. User accounts are copied with their password hashes intact, so existing users keep logging in with the same credentials — no forced resets.

What exactly gets migrated?

Your public database schema and rows, row-level-security policies, auth users, and storage buckets + files. Edge function code and auth-provider settings aren't auto-deployed — a checklist walks you through those.

Is my source project safe?

Yes — the migration is read-only on your Lovable project and only inserts into your new, empty Supabase project. Keep both running until you've tested and are ready to switch.

Can I migrate other platforms?

Lovable Cloud → Supabase is fully supported today. Other sources (Bolt, Replit, Cursor, GitHub) and targets are on the roadmap. Once your data is on Supabase, you can turn the app into real native iOS/Android/Mac/Windows apps with Nativize Studio.

Is this included in Max?

Yes. Migration is included with Nativize Max. If you're not on Max, you can buy a single one-time migration instead.
