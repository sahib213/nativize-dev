Support

Use the support form on the home page, or include the details below when contacting support from the Chrome Web Store listing.

Before requesting support

Please include:

  • The web app or repository you are trying to convert.
  • Whether you used download-only, GitHub push, build, or store-upload features.
  • Any visible error message from the Nativize panel.
  • Whether the issue is with GitHub authentication, generated project files, GitHub Actions, billing, or app-store upload setup.

Do not share secrets

Do not share GitHub tokens, Stripe account secrets, Apple API keys, Google Play service-account JSON, or other credentials in support requests.

Submit a request

Use the support form on the Nativize website. Website support and feature-request submissions are stored in the Nativize Supabase project.
