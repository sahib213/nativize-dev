FAQ

Before you move anything.

Do you store my keys?

No raw secret values are stored. ZIP scanning runs locally and saved scan data contains paths, counts, public project refs, and secret names only. Never paste a service role key.

Does this fully automate migration?

No. The supported Supabase path generates a real plan, safe commands, and tests, while you review and run commands locally. It does not silently modify production.

What platforms are supported?

Lovable Cloud and ZIP sources to Supabase are fully supported. AWS, Firebase, Vercel, Neon, Docker, and generic PostgreSQL use guided checklists where available. Railway and Render are coming soon.

What is Guided Mode?

A real platform-specific checklist based on your scan, without claiming deployment automation that is not safely implemented.

Can I use AWS?

Yes, in Guided Mode. You receive architecture, RDS, S3, Lambda, secrets, DNS, cost, and security checklists.

Can I migrate from Lovable to Supabase?

Yes. That is the first fully supported path, including migrations, edge functions, secrets, storage, auth, integrations, and post-migration tests.

Is this included in Max?

Yes. Active Max subscribers can create full migration projects without consuming single-migration credits.

What happens if my app is complex?

The scanner labels size and risk, surfaces integration-specific work, and keeps the source untouched until you complete the test checklist.
