# Nativize Support

For help with Nativize, use the support form on the Nativize website or the support information on the Nativize Chrome Web Store listing after the listing is published.

Before requesting support, please include:

- The web app or repository you are trying to convert.
- Whether you used download-only, GitHub push, build, or store-upload features.
- Any visible error message from the Nativize panel.
- Whether the issue is with GitHub authentication, generated project files, GitHub Actions, billing, or app-store upload setup.

Do not share secrets such as GitHub tokens, Stripe account secrets, Apple API keys, Google Play service-account JSON, or other credentials in support requests. Website support and feature-request submissions are stored in the Nativize Supabase project.
